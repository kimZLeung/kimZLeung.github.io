关于Getter的疑惑

事情是这样的：今天看了一下Vuex的文档，发现Getters这个属性挺好用的，就是里面回调函数的参数好像有点奇怪，Getters里面的回调接受state为第一个参数，接受getters为第二个参数，getters指向store.getters对象。恩不错，挺好的。 See code ↓↓↓

getters.doneTodos难道不是指向那个函数吗？这不是需要返回getters.doneTodo().length吗？而且getters直接在Vue对象里面计算属性的用法也是这样用，难道？

Vuex文档里面说mutation里面的方法不应该是异步的，因为这样做会增加调试难度。并且同时commit两个mutation的时候会存在不清楚哪个回调函数先调用的问题
所以引入action作为异步commit mutation的基础。

  • 从commit和dispacth的源码中看出…其实commit和dispatch的区别就是后者传入对应参数然后调用并返回了Promise对象，前者是直接取到创建Store时候传入的mutation然后传入对应参数调用而已

不过mutation的触发可以用于切实观察state的状态变化（配合dev-tool打印的日志），所以约定俗成以action管理异步commit，然后用commit同步修改state

通过以上代码也可以看出，state是分模块存放的，getter、mutation、action是全局保存在store的实例里面的。而且getter是通过一个Vue实例的计算属性来读取数据的，再用数据劫持的方法修改getter的get方法。

Yep, Vuex~
