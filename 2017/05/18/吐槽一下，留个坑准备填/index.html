吐槽一下，留个坑准备填

发表于 2017-05-17

话说最近有把Vue拾起来做一个移动端页面的项目。目前只是写两个小页面，就只用抓起Vue就开始用，因为页面的数据简单，所以也没有引入Vuex，也因为不需要路由跳转所以不做Vue-router。我上一次学习Vue起步是在几个月前。也隔得不久，貌似那时候起步的是1.x版本。最近重新自己搭个架子开始做使用了2.x版本的Vue。不得不说更新得比较快。而且挺多API都变了。

render方法的加入，使得用templates属性在Vue实例上挂载组件会报错。（其实这个因果关系也毫无逻辑，因为Vue2.x版本之后，Vue的解析分成了两段，一段是编译时的，一段是运行时的。）其实升级到2.0之后，我们从Vueimport进来的是Vue.common.js（就是编译时的那段）。如果只是使用这个JS的话，并不能使用templates属性来挂载组件。

可以通过在webpack打包的时候使用alias来重命名Vue包里面的内容。也可以通过直接使用render方法来挂载组件。其实这样import进来编译时的vue.common.js是一种优化，所以我还是乖乖地用了render方法

组件上的`$dispatch和$broadcast移除。因为没有使用Vuex这个状态管理的工具。所以子组件向父组件的传递信息只能自己想办法，以前我是利用广播事件来传递数据的。2.x只能使用$emit和$on来管理事件了。除此之外有很多的事件修饰符和按键修饰符，也是方便开发锦上添花吧

其实一开始写这个，只是想写两行吐槽一下技术更新得太快，学生党还得做一些奇奇怪怪的需求所以总是，力不从心。吐槽完还要继续，埋个坑看看什么时候有空过来填一填。。 = =
