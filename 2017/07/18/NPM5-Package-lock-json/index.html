NPM5 - Package-lock.json

发表于 2017-07-18

疑惑

事情是这样的，我之前尝试使用React的Next和Vue的Nuxt写着玩的时候，发现npm install安装依赖之后，项目根目录下除了package.json还多了一个package-lock.json文件，当时有点疑惑但是后来事情做就没去查，今天到服务器上部署项目安装依赖的时候，又发现项目根目录多了个package-lock.json...奇怪了我自己搭起来的东西，也没配置什么别的选项，为什么出现这个文件呢

了解

通过了解原来这个文件是npm5的新东西。每当通过npm安装依赖，即改变node_modules目录之中的依赖时，都会自动生成这个package-lock.json文件，这个文件的主要作用是lock。

也就是把项目的依赖的版本lock起来，让别的机器install依赖的时候，不会安装了最新更新的依赖的比较新的版本。而且lock文件还有一个作用，就是加快依赖的安装速度。

为什么依赖的安装速度lock起来就会变快？因为这个lock文件是这样的

我们可以清楚地看到，这个package-lock.json文件记录了整个node_modules的依赖关系，就是本项目下依赖到的包的详细信息，每一个dependencies的版本，下载地址都详细记录在这个文件里面，所以我们只要有这个文件，安装依赖只需要根据这个文件里面的地址进行下载就好了。

坑

那package-lock.json，给我们带来的安装速度和依赖版本的稳定性之余，也当然给我们带来了一些坑。

  • 从前有一种操作就是：想要更新一个依赖的版本，可以直接在package.json里面修改依赖的版本号，然后直接npm install来安装最新的依赖
  • 但是有了package-lock.json就不可同日而语了，还记得我们前面讲过package-lock.json会锁定依赖的版本嘛？对的，锁定了之后，在package.json里面修改依赖的版本号，也不会更新package-lock.json里面的东西哦。
  • 只能直接通过npm install xxx@版本号来直接安装新版本的模块，通过直接修改node_modules里面的依赖版本从而来修改package-lock.json，这样才能安装到不同版本的依赖，并且同时修改package.json
  • [x] npm 5.3.0 亲测 ... 修改package.json里面的版本号再运行npm install可以更新依赖

自己尝试了一下：

  • 在一个目录下npm init
  • 然后npm install react --save安装react，生成了package.json和package-lock.json两个文件
  • 然后我把package.json里面的dependencies去掉，把node_modules删掉，package-lock.json不作任何修改
  • 再次运行npm install，发现react也不安装了
  • 然后我尝试把react放回去package.json里面，把package-lock.json里面的东西去掉
  • 然后react不装了。

自己的感受是：package-lock.json是用于锁定依赖版本并且可以加快依赖安装速度的，但是安装的依赖依旧是由package.json说了算
